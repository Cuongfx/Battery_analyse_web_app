Battery Life Prediction

Lightweight ML models predict a Remaining-Useful-Life (RUL) class from a cell's cycling data. Results are saved under result/RUL/<file>/.

Lightweight machine-learning models ingest a single or a few most recent cycles and optionally the early-life cycles to output an actionable RUL state (early life → near end of life) at any query point across cells, so they work for both short- and long-lifespan cells. The neural-network model with 16 cycles of input reaches 83.6% accuracy (80.1% for a random forest with single-cycle input) on the MIT–Stanford test set. To alleviate overfitting, covariance matrix adaptation–evolution strategy (CMA-ES) fine-tuning after gradient-based training reduces overfitting and raises accuracy to 85.8%.

Model architecture

A dual-branch network fuses the shape of the degradation with hand-crafted health indicators over a 16-cycle input window:

  • ΔQ branch. Per-cycle ΔQ discharge curves (ΔQ = Qcyc − Qref, 1000 voltage bins) pass through three 1-D Conv-BN-ELU blocks and adaptive average pooling → a 32-D embedding per cycle.
  • Feature branch. 12 hand-crafted summary features per cycle (capacity, charge/discharge time, dQ/dV statistics, log-variance terms …) are linearly projected + ELU to 32-D.
  • Fusion → Bi-GRU. The two branches are concatenated (16 × 64) and read by a 2-layer bidirectional GRU that models the cycle-to-cycle trajectory.
  • Head. Dense layers + softmax over the 5 RUL classes.

RUL classes & actionable states

Remaining Useful Life (cycles until 80% end-of-life) is discretised into five ordered states. As degradation increases, the cell steps down through the classes — the jump into Class 4 is an actionable BMS trigger where degradation accelerates from active-material loss.

  • Class 0 — Early life (RUL > 400)
  • Class 1 — Mid-early life (300 < RUL ≤ 400)
  • Class 2 — Mid life (200 < RUL ≤ 300)
  • Class 3 — Late life (100 < RUL ≤ 200)
  • Class 4 — Near end of life (RUL ≤ 100)
